get_orbital_energy(*args, origin: tuple | list | ndarray = (0, 0, 0), filter_unbound: bool = True, **kwargs)[source]

Obtain the sum of the orbital energy of each particle about some origin, where origin (0, 0, 0) is the center of mass. Two metrics are returned: the orbital energy in the \(\hat{\phi}\) direction and the \(\hat{\theta}\) direction, where \(\phi\) is the azimuthal angle and \(\theta\) is the polar angle. The orbital energy of a single particle is its kinetic energy in some given direction \(mv^2/2\).

We obtain the velocities by first writing the velocity in spherical coordinates

\[\vec{v} = v_r\hat{r} + r\dot{\theta}\hat{\theta} + \dot{\phi}r\sin\phi\hat{\phi}\]

In this way, the velocity components are \(v_\theta = r\dot{\theta}\) and \(v_\phi = \dot{\phi}r\sin\phi\). To find the angular velocities \(\dot{\theta}\) and \(\dot{\phi}\), we use their spherical coordinate relations \(\cos\theta = z/r\) and \(\tan\phi = y/x\). Differentiating each with respect to time yields

\[\dot{\theta} = \frac{r^\prime}{r^2} \left( \frac{zx v_x + zy v_y}{{r^\prime}^2} - v_z\right)\]

where \(r^\prime = (x^2 + y^2)^{1/2}\), and \(\dot{\phi} = (x v_y - y v_x) / {r^\prime}^2\). Therefore,

\[\begin{split}\begin{align} v_\theta &= \frac{y}{r} \left[ \frac{zx v_x + zy v_y}{{r^\prime}^2} - v_z \right]\\ v_\phi &= \frac{r}{{r^\prime}^2} (x v_y - y v_x) \end{align}\end{split}\]

The corresponding orbital energies of a particle are:

\[\begin{split}\begin{align} E_{\mathrm{orb},i,\theta} &= \frac{1}{2} m_i v_{\theta,i}^2\\ E_{\mathrm{orb},i,\phi} &= \frac{1}{2} m_i v_{\phi,i}^2 \end{align}\end{split}\]
Parameters:

  • origin (tuple, list, numpy.ndarray, default = (0, 0, 0)) – The origin relative to the center of mass.

  • filter_unbound (bool, default = True) – If True then only bound particles are used.

  • *args – Positional arguments are passed directly to get_output()

  • **kwargs – Other keyword arguments are passed directly to get_output()

Returns:

  • list – The total orbital energy \(E_{orb,\theta}\) summed over every particle. If filter_unbound is True then an output that has no bound particles has a value of 0.

  • list – The total orbital energy \(E_{orb,\phi}\) summed over every particle. If filter_unbound is True then an output that has no bound particles has a value of 0.

get_period(output: Output | OutputIterator, threshold: float = 0.01)[source]

Obtain the orbital period, if there exists an orbit in the given output file(s). The orbit period is calculated relative to both the primary and secondary stars, using get_primary_IDs() and get_secondary_IDs() while ignoring ejecta. The mean tangential velocities are obtained relative to the overall center of mass (also ignoring ejecta). The orbital period is found as \(P = 2\pi r/v\) for both stars, and then compared according to threshold.

Parameters:

  • output (Output, OutputIterator) – The output or outputs to obtain the orbital period for.

  • threshold (float, default = Pref('threshold', 0.01)) – The relative tolerance used when comparing the two period values obtained from the stars. The comparison proceeds as \(2|P_1 - P_2|/(P_1 + P_2) < \mathrm{threshold}\).

Returns:

If a single output was given as output, returns a single value as the orbital period. Otherwise, returns a list of Unit objects for each given Output in the output iterator.

Return type:

Unit or list

See also

get_period(), get_primary_IDs(), get_secondary_IDs()

get_plunge_time(threshold_frac: int | float | None = None, threshold: int | float | None = 0.05)[source]

Obtain the plunge-in time for a dynamical simulation which originated from a binary simulation. The plunge time here is defined as the time at which some amount of mass from the secondary star has entered a distance from the primary equal to the primary’s original SPH radius (from relaxation).

Parameters:

  • threshold_frac (int, float, None, default = Pref('get_plunge_time.threshold_frac', None)) – The fraction of the secondary’s total starting mass that must be within the secondary’s initial radius to identify the plunge-in event.

  • threshold (int, float, None, default = Pref('get_plunge_time.threshold', 0.05)) – The mass in solar masses of the secondary star which must be within the primary’s starting radius in order to detect a plunge-in event.

Returns:

The plunge time. Returns None if there is not yet any plunge-in event in this simulation.

Return type:

Unit or None
