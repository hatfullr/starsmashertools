class EffGravPot(m: list | tuple | ndarray, x: list | tuple | ndarray, y: list | tuple | ndarray, z: list | tuple | ndarray, period: int | float = 0.0, G: int | float | None = None)[source]

A class for working with effective gravitational potentials in an Output, which can come from either a Binary or from a Dynamical prior to the plunge-in. In the presence of a binary in which the individual stars are represented as point masses, an individual fluid packet of mass \(m\) experiences a force \(F=-m\nabla\varphi\), where

\[\varphi(\vec{r}) = -\frac{GM_1}{r_1} - \frac{GM_2}{r_2} - \frac{1}{2}\omega^2r_\mathrm{com}^2\]

in which \(G\) is the gravitational constant, \(M_1\) and \(M_2\) are the masses of stars 1 and 2, \(r_1=|\vec{r}-\vec{r}_1|\) and \(r_2=|\vec{r}-\vec{r}_2|\) are the distances of the fluid packet from stars 1 and 2, \(\omega\) is the oribtal frequency, and \(r_\mathrm{com}=|\vec{r}-\vec{r}_\mathrm{com}|\) is the distance of the fluid packet from the center of mass.

To obtain \(\varphi(\vec{r})\) without assuming the stars are point masses, specify as_point_masses = False in __init__(). Here,

\[\varphi(\vec{r}) = -\sum_i^N\frac{Gm_i}{r_i} - \frac{1}{2}\omega^2r_\mathrm{com}^2\]

where \(i\) represents the SPH particle index and \(r_i=|\vec{r}-\vec{r}_i|\) is the distance between \(\vec{r}\) and particle \(i\). This method is significantly slower.

Class constructor.

Parameters:

  • m (list, tuple, numpy.ndarray) – Masses of the particles. Must be 1D.

  • x (list, tuple, numpy.ndarray) – x-positions of the particles. Must be 1D and have the same length as m.

  • y (list, tuple, numpy.ndarray) – y-positions of the particles. Must be 1D and have the same length as m.

  • z (list, tuple, numpy.ndarray) – z-positions of the particles. Must be 1D and have the same length as m.

  • period (int, float, default = 0.) – The oribtal period of the binary system.

  • G (int, float, Unit, None, default = None) – The value of the gravitational constant. If None, the value stored in the constants in units is used. If a Unit is given, it is converted to a float.

get_gradient(positions: list | tuple | ndarray)[source]

Sample the effective gravitational potential gradient \(\nabla\varphi\) field at the given positions. This is the same as negative the gravitational acceleration vector, since \(m\vec{g}=\vec{F}=-m\nabla\varphi\).

Parameters:

positions (list, tuple, numpy.ndarray) – The x, y, and z positions to get the effective gravitational potential gradient at. Must have a shape (n,3).

Returns:

The gradient of the effective gravitational potential at positions (x, y, z). The result is an array of 3-vectors, with the same length as x, y, and z.

Return type:

numpy.ndarray

get_lagrange(which: list | tuple | ndarray = [1, 2, 3], extent_frac: int | float = 5.0)[source]

Get the n’th Lagrange points.

Parameters:

  • which (list, tuple, numpy.ndarray, default = [1,2,3]) – The Lagrange points to obtain. Currently only \(L_1\), \(L_2\), and \(L_3\) are supported.

  • extent_frac (int, float, default = 5.) – The fraction of the maximum extent to search. The extent is the distance of the farthest-away particle. We search beyond that particle by this amount to sample \(\varphi\).

Returns:

A NumPy array of shape (n,3) with the x, y, and z positions of the Lagrange points where n is the length of which.

Return type:

numpy.ndarray

rotate(x: list | tuple | ndarray, y: list | tuple | ndarray, z: list | tuple | ndarray, xangle: float | int | float64 | integer = 0, yangle: float | int | float64 | integer = 0, zangle: float | int | float64 | integer = 0)[source]

Rotate the given x, y, and z points using an Euler rotation.

An Euler rotation can be understood as follows. Imagine the x, y, and z axes as wooden dowels. First the z-axis dowel is rotated zangle degrees clockwise, then the y-axis dowel is rotated yangle degrees clockwise, and finally the x-axis dowel is rotated xangle degrees clockwise.

Parameters:

  • x (list, tuple, numpy.ndarray) – The x components.

  • y (list, tuple, numpy.ndarray) – The y components.

  • z (list, tuple, numpy.ndarray) – The z components.

  • xangle (float, int, numpy.float_, numpy.integer, default = 0) – The x component of an Euler rotation in degrees.

  • yangle (float, int, numpy.float_, numpy.integer, default = 0) – The y component of an Euler rotation in degrees.

  • zangle (float, int, numpy.float_, numpy.integer, default = 0) – The z component of an Euler rotation in degrees.

Returns:

A copy of the x, y, z components which were given as inputs, rotated by xangle, yangle, and zangle.

Return type:

x, y, z
